get /myResources

Algumas coleções precisam de paginação no método GET.

  • Request
  • Response

Headers

  • REST-Range: required (string)

    Descreve qual parte da coleção deseja-se recuperar nesta consulta. O conteúdo desse cabeçalho deve seguir o formato "resources={from}-{to}". O primeiro item da coleção é o item 0.

    Example:

    resources=0-49

HTTP status code 200

Quando a resposta retornada para uma coleção corresponder à coleção completa, mesmo que tenha sido solicitada uma parte específica, o serviço retornará o status 200.

Example:

GET /users
REST-Range: resources=0-49


200 OK
REST-Accept-Ranges: resources
REST-Content-Range: resources 0-33/34

Headers

  • REST-Accept-Ranges: required (string)

    Informa qual tipo de range o recurso aceita. O único valor definido por esta especificação é "resources".

    Example:

    REST-AcceptRanges: resources

  • REST-Content-Range: required (string)

    Informa qual a parte da coleção retornada nesse response. No caso do status 200, ele sempre indicará que a parte é composta do primeiro ao último item, ou seja, a coleção completa.
    O conteúdo desse cabeçalho segue o formato "resources 0-{last}/{total}".

    Example:

    REST-Content-Range: resources 0-33/34

  • Vary: required (string)

    É importante que o header Vary esteja presente na resposta à requisição com REST-Range, para garantir que o cache seja feito de maneira correta, uma vez que Ranges diferentes determinam respostas diferentes.
    Uma resposta a um Range PRECISA incluir o campo REST-Range no header Vary.

    Example:

    Vary: REST-Range,Accept

HTTP status code 416

Este código é retornado quando o incício da faixa solicitada está além do total de recursos disponíveis.

Example:

GET /users
REST-Range: resources=50-99


416 Requested Range Not Satisfiable
REST-Accept-Ranges: resources
REST-Content-Range: resources */34

Headers

  • REST-Accept-Ranges: required (string)

    Informa qual tipo de range o recurso aceita. O único valor definido por esta especificação é "resources".

    Example:

    resources

  • REST-Content-Range: required (string)

    Informa quantos itens estão disponíveis para consulta. O conteúdo desse cabeçalho segue o formato "resources */{total}"

    Example:

    REST-Content-Range: resources */157

  • Vary: required (string)

    É importante que o header Vary esteja presente na resposta à requisição com REST-Range, para garantir que o cache seja feito de maneira correta, uma vez que Ranges diferentes determinam respostas diferentes. Uma resposta a um Range PRECISA incluir o campo REST-Range no header Vary.

    Example:

    Vary: REST-Range,Accept

Headers

  • Content-Version: required (string)

    Indica a versão da entidade criada no servidor.

    Example:

    Content-Version: 21FFEC

  • Content-Latest-Version: required (string)

    Indica a versão mais recente da entidade disponível no servidor. Geralmene traz a mesma versão trazida em Content-Version.

    Example:

    Content-Latest-Version: 21FFEC
